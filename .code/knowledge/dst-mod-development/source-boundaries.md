# 饥荒 MOD 来源与协议边界

## 允许记录

- Workshop MOD 名称、ID、公开页面链接。
- MOD 功能行为描述。
- 配置项含义和效果。
- 兼容性观察。
- 日志、错误现象和测试结论。
- 自己重写后的设计、接口和代码。

## 不允许记录

- 第三方 MOD 的完整源码。
- 明确禁止再分发的代码片段。
- 未授权复制的 UI、素材、图片、音频或文案。
- 真实 Klei token、服务器密码、玩家 ID、存档、Steam 缓存或 Workshop 下载包。

## 可参考但需重写

- Insight / Show Me+：可参考信息分层、RPC、descriptor、客户端 UI 思路；不直接复制源码。
- 资源、保鲜、复活、操作增强类 Workshop MOD：可记录行为和冲突点；实现时重写模块。
- 现有服务器 MOD 组合：可作为需求来源和黑盒验证对象，不作为源码复制对象。

## 公开分发前要求

后续开源前至少需要补齐：

- `LICENSE`
- `NOTICE`
- 中文 README
- 英文 README
- 第三方来源说明
- Workshop MOD 行为参考清单
- 不包含第三方受限源码的自查记录

## 判断规则

如果一个能力只能通过复制第三方实现才能完成，先停止开发，改为：

1. 记录目标行为。
2. 记录观察到的输入、输出和配置效果。
3. 设计自己的数据结构和模块边界。
4. 在测试世界用黑盒行为验证。
5. 重写实现。
