# 饥荒 MOD 开发知识库

## 目标

本知识库用于沉淀《饥荒联机版》MOD 开发知识，指导后续资源减负、服务器管理、游戏内辅助和管理端联动开发，避免无依据、无验证、无边界的开发。

这里记录的是开发前必须理解的机制、来源边界、风险和验证方法，不是随手堆功能的需求池。

## 开发原则

- 先理解 DST MOD 机制，再写代码。
- 先做低风险、可开关、可回滚能力。
- 任何会改变世界状态的功能都必须默认保守。
- 不直接复制第三方 MOD 源码，只记录行为观察、配置效果和重写思路。
- 管理端、服务端 Lua、客户端 UI、RPC 权限要分层设计。
- 每个能力都要说明适用场景、默认值、影响范围和验证方式。

## 风险分级

- L1：只读信息、配置展示、日志分析。
- L2：耐久、保鲜、返还比例、动作速度等可局部回滚能力。
- L3：生成实体、删除实体、改变掉落倍率、资源再生。
- L4：修改 Boss 血量、跨分片状态、管理员命令、RPC 控制、游戏内 UI 权限。

L3 和 L4 不能直接开写，必须先写设计说明、测试计划和回滚方式。

## 优先沉淀方向

第一阶段：低风险机制和小能力。

- `modinfo.lua` 配置项写法。
- `modmain.lua` 模块加载方式。
- 暖石、帐篷、工具耐久。
- 保鲜策略。
- 建筑返还。
- 灭火器策略。

第二阶段：操作增强。

- 快速采集。
- 快速动作。
- 自动门。
- 掉落物堆叠。

第三阶段：资源减负。

- 采集倍率。
- 矿石多次开采。
- 世界资源再生。
- 实体清理策略。

第四阶段：服务器管理与联动。

- 聊天命令。
- 管理员工具。
- 游戏内面板。
- 与 `dst-admin` 管理端联动。

## 开发前检查

每次开始一个 MOD 能力前，先回答：

- 这个能力解决什么具体游玩负担？
- 它属于哪个风险等级？
- 是否影响 Master、Caves，还是两边都影响？
- 是否需要客户端代码或 RPC？
- 是否已有 Workshop MOD 提供类似能力？
- 参考来源是否允许复制，还是只能重写？
- 开启、关闭、回滚分别怎么验证？
