# 饥荒 MOD 开发技能卡

## 适用场景

当任务涉及 DST MOD 开发、MOD 整合、资源减负、Lua 模块、游戏内 UI、RPC、管理员命令或 `dst-admin` 与 MOD 联动时，先阅读本文件。

## 执行流程

1. 定义能力
   - 写清解决的具体游玩问题。
   - 标记风险等级：L1、L2、L3、L4。
   - 标记影响范围：Master、Caves、客户端、管理端。

2. 查机制
   - 明确需要用 `modinfo.lua`、`modmain.lua`、PostInit、RPC、network variable 还是 UI。
   - 不确定 DST API、生命周期或分片行为时，先查资料或做最小实验。

3. 查来源
   - 查看 `.code/knowledge/dst-mod-development/source-boundaries.md`。
   - 只记录行为和配置，不复制受限源码。
   - 如参考 Workshop MOD，记录名称、ID、能力和重写策略。

4. 设计接口
   - 每个能力必须有独立开关。
   - 高风险能力默认关闭。
   - 涉及管理员操作时必须有权限检查。
   - 涉及世界状态变更时必须有日志和回滚说明。

5. 最小实现
   - 先做一个模块、一个开关、一个可验证能力。
   - 不一次性整合多个高风险能力。
   - 不把真实服务器作为首个测试环境。

6. 验证
   - 在测试世界验证无 Lua 报错。
   - 验证开启和关闭两种状态。
   - 验证 Master/Caves 分片影响。
   - 对删除、生成、倍率、血量类能力记录回滚方式。

## 输出要求

每个 MOD 能力至少留下：

- 模块目标。
- 默认值。
- 配置项。
- 影响范围。
- 来源参考。
- 验证方式。
- 回滚方式。

## 禁止事项

- 不凭想象写 DST API。
- 不默认启用删除实体、资源再生、倍率增强或 Boss 血量修改。
- 不直接复制第三方 MOD 源码。
- 不把玩家 ID、服务器密码、token 写入仓库。
